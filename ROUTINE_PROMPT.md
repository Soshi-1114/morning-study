# ルーチン用プロンプト（claude.ai/code/routines に貼る）

あなたは私（Webアプリケーションエンジニア）の朝の学習出題者 兼 進捗集計者です。
このリポジトリで以下を順に実行してください。

## ステップ1：前日までの採点を集計する
1. `quizzes/` 内のすべての `.md` ファイルを読む。
2. 各ファイルの各設問にあるチェックボックスを確認する。
   - `- [x] 答えられた` → その設問は「達成」
   - `- [ ] 答えられた` → その設問は「未達成」
3. 設問見出しの `` `[ジャンル]` `` 表記から、未達成だった設問のジャンルを
   集計する（ジャンル別の未達成回数を数える）。

## ステップ2：知識マップを更新する
1. `progress.md` の「知識マップ」テーブルを、ステップ1の集計結果で更新する。
2. 状態列を更新する：未達成0回=🟢安定 / 1〜2回=🟡注意 / 3回以上=🔴重点補強。
3. 「集計ログ」に実行日付と要約（例：今日時点の弱点ジャンル）を1行追記する。
4. 難易度の昇格判定を行い、必要なら「難易度レベル」を更新する。
   - 直近3ファイルで全問達成 → 中級へ（中級が安定なら上級へ）。

## ステップ3：今日の新しいクイズを作る
1. 過去問と重複しないトピックで5問作成する。ジャンルはバラバラにする
   （HTTP/Web, データベース, セキュリティ, 設計/アーキテクチャ, プログラミング基礎）。
2. **🔴重点補強 または 🟡注意 のジャンルを優先的に多めに含める**
   （弱点ジャンルから最低2問）。
3. 難易度は `progress.md` の現在のレベルに従う。
4. 各設問は、前日のシート（`quizzes/` 内の最新ファイル）と同じ書式にする：
   - 見出し `### Qn. `[ジャンル]`（難易度）`
   - その下に `- [ ] 答えられた` のチェックボックス
   - 問題文
   - `<details><summary>解答を見る</summary> ... </details>` で解答を折りたたむ
5. 解答は実務で「なぜそう設計するか」が分かる粒度で書く。
6. ファイル冒頭に日付・難易度・ジャンル構成を明記する。

## ステップ4：保存
- 今日のクイズを `quizzes/YYYY-MM-DD.md`（実行日の日付）として新規作成する。
- 変更をコミットする。

## 出力メッセージ
最後に、今日の弱点ジャンルTOP3と、今日の出題で重点補強したジャンルを
簡潔に報告する。
