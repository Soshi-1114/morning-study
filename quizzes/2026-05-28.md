### Q1. `[HTTP/Web]`（初級）

- [ ] 答えられた

HTTPメソッド GET・POST・PUT・DELETE のそれぞれの用途と、「冪等性（べきとうせい）」の観点からの違いを説明してください。

<details>
<summary>解答を見る</summary>

| メソッド | 用途 | 冪等か |
|--------|------|------|
| GET | リソースの取得 | ✅ 何度呼んでも同じ結果 |
| POST | リソースの新規作成 | ❌ 呼ぶたびに新しいレコードが増える |
| PUT | リソースの完全置換 | ✅ 同じデータを何度送っても結果は同じ |
| DELETE | リソースの削除 | ✅ 2回目以降は「すでに存在しない」で結果は同じ |

**なぜ大事か：** 冪等なメソッドはリトライしても副作用がないため、ネットワーク障害後に安全に再送できる。POSTのリトライは二重決済などのバグに直結する。

</details>

### Q2. `[データベース]`（初級）

- [ ] 答えられた

データベースの「インデックス」とは何か、使うメリットとデメリットをそれぞれ説明してください。

<details>
<summary>解答を見る</summary>

**インデックスとは：** 特定のカラムの値と行の場所を対応付けた索引。本の巻末索引と同じ仕組みで、全行を舐めなくても目的の行に素早くたどり着ける。

**メリット：**
- SELECT のパフォーマンスが劇的に改善する（O(n) → O(log n) など）
- WHERE・JOIN・ORDER BY の対象カラムに効く

**デメリット：**
- INSERT / UPDATE / DELETE のたびにインデックスも更新が必要 → 書き込みが遅くなる
- インデックスデータ自体がディスク容量を占有する

**実務のコツ：** 検索頻度が高く、かつ絞り込み効果が高い（カーディナリティが高い）カラムに貼る。むやみに増やすと書き込みボトルネックになる。

</details>

### Q3. `[セキュリティ]`（初級）

- [ ] 答えられた

XSS（クロスサイトスクリプティング）とはどのような攻撃か、また基本的な対策を説明してください。

<details>
<summary>解答を見る</summary>

**XSSとは：** 攻撃者がWebページに悪意のあるスクリプト（JavaScript）を埋め込み、閲覧したユーザーのブラウザ上でそのスクリプトを実行させる攻撃。

**被害例：**
- セッションCookieの盗取 → アカウントなりすまし
- フィッシング用の偽フォーム表示
- ユーザーの操作の乗っ取り

**基本的な対策：**
1. **出力時エスケープ**：HTMLに埋め込む際に `<` → `&lt;` などを変換する（根本的な対策）
2. **Content Security Policy (CSP)**：信頼できるスクリプトソースのみ許可するHTTPヘッダ
3. **HttpOnly Cookie**：JavaScriptからCookieを読めないようにする（被害を限定化）

Reactなどのモダンフレームワークはデフォルトでエスケープしてくれるが、`dangerouslySetInnerHTML` などの抜け穴には注意。

</details>

### Q4. `[設計/アーキテクチャ]`（初級）

- [ ] 答えられた

SOLID原則の「S」である単一責任原則（SRP: Single Responsibility Principle）を説明し、違反するとどうなるか例を挙げてください。

<details>
<summary>解答を見る</summary>

**SRPとは：** 「クラス（or モジュール）を変更する理由は1つだけであるべき」という原則。

**違反例：** `UserService` というクラスが「ユーザー登録ロジック」「メール送信」「ログ書き込み」を全部担っているとする。
- メール業者を変えたくなった → UserServiceを触る必要がある
- ログ形式を変えたくなった → UserServiceを触る必要がある
- ユーザー登録仕様が変わった → また同じファイルを触る

→ 1つのクラスを変える理由が複数あるため、変更のたびに無関係な機能を壊すリスクが生まれる。

**正しい分割：** `UserRegistrationService`・`EmailService`・`Logger` に分ける。それぞれが独立して変更できる。

</details>

### Q5. `[プログラミング基礎]`（初級）

- [ ] 答えられた

スタック（Stack）とキュー（Queue）の違いを、データの出し入れの順番と実用例を挙げて説明してください。

<details>
<summary>解答を見る</summary>

| | スタック | キュー |
|--|---------|-------|
| 取り出し順 | **LIFO**（後入れ先出し） | **FIFO**（先入れ先出し） |
| イメージ | 積み重ねた皿（一番上から取る） | 行列（先に並んだ人が先に通る） |

**スタックの実用例：**
- 関数呼び出しのコールスタック（最後に呼ばれた関数が最初に返る）
- ブラウザの「戻る」ボタン（最後に訪れたページに戻る）
- アプリのundo/redo

**キューの実用例：**
- メッセージキュー・ジョブキュー（先にリクエストが来た順に処理）
- プリンタの印刷待ち
- BFS（幅優先探索）

</details>
